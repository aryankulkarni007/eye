# M5 - match codegen (Strategy A)

> **Historical design brief** for v0.3 milestone M5 (complete). Canonical
> status, limitations, and test map: [`FUTURE.md`](FUTURE.md). Active work:
> v0.5 in that file. Keep this document for hoist algorithm and block-boundary
> rules.

Brief for lowering `Expr::Match` to C. This was the last piece of compiler
work for v0.3; M6 is the end-to-end test that proves it.

## Goal

Turn a lowered `Expr::Match { scrut, arms }` into C. Two shapes:

1. **Statement-position match** (`match sh { ... };` as an expression
   statement) lowers directly to a `switch`, no temporary.
2. **Non-statement-position match** (anything that produces a value: a `let`
   initializer, a call argument, an operand) is hoisted: declare a temp at the
   top of the nearest enclosing statement, fill it with a `switch`, then
   substitute the temp at the use site.

This is a uniform rule with no GCC statement-expression extension.

## What M4 already gave us

`crates/hir/src/core.rs` records the match expression's type in
`body.expr_types` as the first arm body's type (mirrors `if` lowering). That
recorded type is the contract that lets codegen declare `Type _matchN;` for the
hoist temp. Without it there is nothing to write on the left of the
declaration. See the constraint section below for the case where it is absent.

The scrutinee is guaranteed to be a known enum (HIR emits a diagnostic and the
match still lowers otherwise, but codegen assumes a clean HIR). Exhaustiveness
is already enforced in HIR: a match with no wildcard covers every variant, so a
generated `switch` either lists every `case` or carries a `default`.

## Current codegen shape (what we are working against)

`crates/codegen/src/core.rs` builds a single growing `output: String` with an
`indent_level`. There is no buffering and no hoist mechanism today:

- `gen_stmt` pushes indent, emits the statement, appends `;\n`.
- `gen_expr` appends expression text inline and recurses into children.
- The `Expr::Match` arm is a placeholder: `self.output.push_str("/* MATCH (M5) */");`.

Because output is linear and top-down, by the time `gen_expr` reaches a match
inside `let int32 n = match sh { ... }` the prefix `let int32 n = ` is
already written. We cannot insert the `switch` before it from inside
`gen_expr`. The fix is to hoist _before_ the statement body is emitted, from
`gen_stmt`.

## Implementation

All changes are in `crates/codegen/src/core.rs`. Nothing in HIR changes.

### 1. New state on `CGen`

```rust
match_temps: FxHashMap<ExprId, String>, // match ExprId -> hoisted temp name
match_counter: usize,                   // monotonic; reset per function
```

`match_temps` is consulted by `gen_expr`; `match_counter` names temps
`_match0`, `_match1`, ... Reset `match_counter` to 0 (and clear `match_temps`)
at the top of `gen_function` so names stay small and local per C function.
The implementation uses `rustc_hash::FxHashMap` (codegen now depends on
`rustc-hash`); a plain `std::collections::HashMap` would do equally well since
the map holds only a handful of entries per function.

### 2. `hoist_matches(&mut self, expr_id, body)` - the prelude pass

Called from `gen_stmt` before the statement body is emitted. It walks the
statement's expression subtree in **post-order** (children first), and for each
match it finds in a non-statement position it:

1. allocates a name `_match{N}` from `match_counter`,
2. emits `Type _match{N};` at the current indent (type from
   `body.expr_types.get(match_id)` run through `map_type_ref`),
3. emits the assigning `switch` (see step 4),
4. records `match_temps.insert(match_id, name)`.

Post-order matters: a match nested inside another match's scrutinee or arm must
be declared and assigned before the outer one references it.

**Block-boundary rule (precise).** The walk recurses through _inline_ child
expressions only, and stops at block-shaped children, because those are emitted
as their own statement contexts (or inline ternaries) and must hoist into their
own scope, not this statement's prelude:

- **Recurse into:** `Match::scrut`, `Binary::{lhs,rhs}`, `Unary::operand`,
  `Call::{callee,args}`, `Field::base`, `Assign::{lhs,rhs}`, `Ref::operand`,
  `Deref::operand`, `StructLit` field values. Also recurse into a `Match`'s own
  scrutinee, then register the match itself.
- **Do not recurse into:** the `then_branch`/`else_branch` blocks of `If`, the
  `body` block of `Loop`, the body of `Expr::Block`, and match **arm bodies**.

So a match in an `if` condition is hoisted (good - the condition is inline),
but a match inside an `if` body is left for that body's own `gen_stmt` pass.

### 3. `gen_stmt` changes

- `Stmt::Let { init: Some(e), .. }`: call `hoist_matches(e, body)` first, then
  emit the declaration as today. The match inside `e` now renders as its temp.
- `Stmt::Expr(e)` where `body.exprs[e]` is `Expr::Match { .. }`: this is the
  statement-position form. Emit a **direct** `switch` (step 4, statement
  variant: arm bodies are statements, no temp). Do not hoist, do not append a
  trailing `;`.
- `Stmt::Expr(e)` otherwise: call `hoist_matches(e, body)` first, then emit as
  today.

### 4. The `switch` emitter

One helper, two variants, driven by whether a temp name is supplied:

```
switch (<scrut>) {
    case <VariantName>: <arm>; break;
    ...
    default: <wildcard arm>; break;   // only if an arm is Pat::Wildcard
}
```

- Scrutinee: `gen_expr(scrut, body)` (normally a `Path` to the enum-typed
  local). Variant case labels: the bare C enum constant name, looked up via
  `self.hir.enums[enum_id].variants[idx].name` (enum constants are global bare
  names in our C backend, so `case Circle:` is valid).
- `Pat::Wildcard` -> `default:`. `Pat::Variant { enum_id, idx }` -> `case <name>:`.
  `Pat::Missing`/`Pat::Bind` cannot appear in a match arm after M4 lowering;
  treat as unreachable.
- **Assigning variant** (hoisted): each arm emits
  `case X: _matchN = <arm body expr>; break;`, wildcard emits
  `default: _matchN = <body>; break;`.
- **Statement variant** (statement-position): each arm emits
  `case X: <arm body expr>; break;`, wildcard `default: <body>; break;`.

`gen_expr`'s `Expr::Match` arm becomes: look up `match_temps.get(&expr_idx)` and
push the temp name. If it is absent that is a codegen bug (a non-statement match
that was never hoisted), not a fallback - it should never happen given step 3.

### 5. Arm body shapes

v0.3 arm bodies are simple value expressions (literals, paths, arithmetic,
calls). Two shapes to document:

- **Simple expr** (the only shape M6 exercises): emit inline as above.
- **Block body** (`Expr::Block`): emit
  `case X: { <stmts>; _matchN = <tail>; break; }` for the assigning form, or
  `case X: { <stmts>; <tail>; break; }` for the statement form. Implementation
  may defer this; note the shape so it is not forgotten.

## The arm-body-type constraint (resolved)

A hoisted match needs `body.expr_types.get(match_id)` to be `Some` so codegen
can write `Type _matchN;`. That entry exists only when the first arm body has a
recorded type. Today calls do not carry a type (`// call return type is unknown
for now.`), so `let x = match sh { _ -> some_call() };` would leave the temp
untyped.

**Decision for v0.3:** the realistic surface (int/float/bool/char/string
literals, typed locals, field access, arithmetic) all carry a type, and the M6
fixture uses int literals, so the constraint holds in practice. When the type is
genuinely absent, codegen falls back to `int32` for the temp and emits a
trailing comment `/* match temp type unknown */` so the output still compiles
and the gap is visible. Do **not** emit `void* _matchN`. This is a documented
v0.3 limitation, not a silent guess.

## Known limitations (v0.3)

- Match inside a ternary-`if` branch is not hoisted (the branch is emitted
  inline by `gen_expr`, and hoisting would set the temp unconditionally,
  changing evaluation semantics). The realistic surface (`let T n = match
...;`) never hits this. Out of scope for v0.3.
- Nested match in an arm body works via the block-body shape but is untested in
  v0.3; the M6 fixture keeps arm bodies to single expressions.
- Arm body whose only type source is a call return value falls back to `int32`
  (see constraint above).
